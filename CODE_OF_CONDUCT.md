# Code of Conduct

## Our Pledge

We aim to keep this project professional, respectful, and evidence-oriented.

Contributors are expected to discuss ideas, code, protocols, and claims constructively. Disagreement is welcome when it is specific, actionable, and grounded in evidence.

## Expected Behavior

- Be respectful and direct.
- Critique claims and implementation details, not people.
- Keep provider, runtime, and capability claims truthful.
- Respect privacy boundaries and do not request private credentials or workspace data.
- Document limitations clearly.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Publishing private information or credentials.
- Pressuring others to expose private runtime state.
- Intentionally overstating unverified provider or runtime capabilities.
- Submitting generated or copied content without appropriate rights.

## Enforcement

Maintainers may edit, close, or remove issues, pull requests, or comments that violate this code of conduct.

For serious concerns, contact the maintainers through a private channel when available.
