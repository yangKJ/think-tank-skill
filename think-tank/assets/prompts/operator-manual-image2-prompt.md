# Image2 Prompt: Operator Manual

Create a realistic documentation visual for the open-source AI skill
"think-tank" that explains how an operator uses mode, policy, capabilities, and
boundaries.

Visual goal:
- make the skill feel like a controllable operating surface
- show decisions, not just decoration

Composition:
- a central control panel labeled "think-tank"
- four surrounding panels labeled "mode", "policy", "capability", "boundary"
- visible route chips for "research", "review", "council", "strategy"
- one capability section with slots like "source-acquisition" and
  "browser-automation"
- a boundary panel showing "selected", "invoked", and "verified_partial" as
  distinct states

Style:
- realistic product UI illustration
- calm light background
- serious developer-tool feeling
- no fantasy effects
- high readability
- structured layout with intentional spacing

Aspect ratio: 16:9.
