# v3.0 Release Notes

v3.0 adds the **Skill Experience Layer**.

This release improves how agents decide when to use `think-tank`, how they form
an invocation contract, how they load references progressively, and how they
compose optional peer skills without overstating provider capability.

## Added

- `protocol/skill-trigger-intelligence.md`
- `protocol/skill-invocation-contract.md`
- `protocol/progressive-disclosure.md`
- `docs/agent-compatibility-matrix.md`
- `docs/skill-composition-guide.md`
- `docs/skill-quality-score.md`
- `examples/v3/`
- `../self-tests/`
- `checks/skill_experience_check.py`

## Changed

- Public trigger phrasing is now explicitly policy-owned. The skill documents
  trigger categories and examples, but user YAML policy owns actual trigger
  words and provider preferences.
- README and docs navigation now include v3 skill experience references.

## Verification Boundary

v3.0 does not claim every optional provider works out of the box.

It claims the public skill has a clearer agent-facing contract for:

- activation decisions
- invocation structure
- progressive disclosure
- composition with peer skills
- compatibility degradation
- self-test fixtures
- quality scoring
