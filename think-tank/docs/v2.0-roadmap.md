# v2.0 Roadmap

v2.0 upgrades think-tank from a protocol-first skill into **Research OS + Memory Runtime**.

The goal is not to add more bundled providers. The goal is to make every serious run replayable, memory-aware, provider-truthful, and safe to extend.

## Scope

```yaml
version: "2.0"
theme: Research OS + Memory Runtime
core_upgrade:
  - run_record
  - project_memory_runtime
  - provider_invocation_ledger
  - handoff_protocol
  - guardrails
  - research_os
  - eval_pack
```

## Feature Map

| feature | protocol | value |
|---|---|---|
| Run Record | [`protocol/run-record.md`](../protocol/run-record.md) | make runs replayable |
| Project Memory Runtime | [`protocol/project-memory-runtime.md`](../protocol/project-memory-runtime.md) | turn useful results into scoped memory candidates |
| Provider Invocation Ledger | [`protocol/provider-invocation-ledger.md`](../protocol/provider-invocation-ledger.md) | prove selected vs invoked provider states |
| Handoff Protocol | [`protocol/handoff-protocol.md`](../protocol/handoff-protocol.md) | structure profile, subagent, provider, and human handoffs |
| Guardrails | [`protocol/guardrails.md`](../protocol/guardrails.md) | protect high-risk provider and memory operations |
| Research OS | [`protocol/research-os.md`](../protocol/research-os.md) | define local `.think-tank/` workspace contracts |
| Eval Pack | [`protocol/eval-pack.md`](../protocol/eval-pack.md) | prevent regression in output structure and boundaries |

## Non-Goals

- Store private project memory in the public skill core.
- Bundle peer providers or MCP servers.
- Claim true multi-agent runtime without runtime evidence.
- Claim provider invocation from policy selection alone.
- Publish user-local `.think-tank/` contents.
