# v2.0 Release Notes

## Summary

v2.0 introduces the Research OS + Memory Runtime layer for think-tank.

This release makes runs more replayable, memory-aware, provider-truthful, and safer to extend with optional providers.

## Added

- **Run Record** protocol, schema, template, and fixture.
- **Project Memory Runtime** protocol, schema, template, and fixture coverage.
- **Provider Invocation Ledger** protocol, schema, template, and fixture.
- **Handoff Protocol** for profile, subagent, provider, and human handoff packets.
- **Guardrails** for permission, privacy, evidence, artifact, memory, and security gates.
- **Research OS** local workspace contract for `.think-tank/`.
- **Eval Pack** contract for regression fixtures.
- `checks/v2_0_release_check.py`.

## Clarified

- Public think-tank core defines contracts; user-local `.think-tank/` stores project data.
- Project memory is proposed, reviewed, and promoted; it is not automatic note-taking.
- Provider ledger entries must distinguish `selected`, `preflight_checked`, `invoked`, `recovered`, and `verified`.
- Handoff examples must disclose whether they are true subagent runtime or single-agent multi-profile flow.
