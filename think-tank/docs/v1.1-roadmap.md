# v1.1 Roadmap

v1.1 focuses on making optional provider usage understandable without turning examples into unsupported capability claims.

## Goals

1. Document provider integration patterns.
2. Add workflow pattern examples that show possible user approaches.
3. Keep English and Chinese README entrypoints synchronized.
4. Govern public Image2 assets and avoid SVG primary visuals.
5. Document Codex installation boundaries.
6. Add a v1.1 release gate.

## Non-Goals

- Bundle peer skills.
- Claim default social, media, browser, or knowledge-base provider invocation.
- Turn current local `.codex/skills/` into a public dependency.
- Expose private project workflow, credentials, outputs, or local provider policy.

## Completion Criteria

```yaml
provider_patterns: documented
workflow_patterns: documented
readme_language_sync: checked
visual_assets: governed
codex_installation: documented
release_gate: v1_1_release_check
```

## Version Boundary

v1.1 is still a protocol-first open-source skill release. It improves onboarding and integration clarity. It does not change the core claim that provider invocation requires environment-specific evidence.
