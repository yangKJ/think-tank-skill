# v3.0 Roadmap: Skill Experience Layer

v3.0 focuses on making `think-tank` easier for Codex, Claude Code, and other
agents to discover, invoke, compose, and validate.

It does not turn `think-tank` into a full runtime product. It improves the
public skill experience layer.

## Goals

- make activation decisions explicit without hard-coding trigger words
- define an invocation contract agents can emit before deep work
- document progressive disclosure so agents load only necessary refs
- explain compatibility across Codex, Claude Code, and generic agents
- document safe composition with optional peer skills
- add a public self-test fixture kit
- add a quality score rubric
- add a release gate for the v3 skill experience layer

## Non-Goals

- no built-in project-specific trigger words
- no hard dependency on any optional peer skill
- no claim that all providers are automatically invoked
- no full cross-platform runtime parity claim
- no private project or local machine state in the public core

## Exit Criteria

v3.0 is complete when:

- README and docs index link to the v3 skill experience layer
- protocol index lists the new protocol files
- examples and self-test fixtures are present
- release suite runs the v3 gate
- privacy and package boundary checks still pass
