# v0.1 Foundation Final

本文是 think-tank `0.1.0` foundation 阶段的最终收口记录。

## 总体结论

```yaml
version: 0.1.0
stage: foundation_final
status: ready_for_review
primary_platform: codex
secondary_platform: claude-code
```

think-tank 已经从旧 research agent 子技能和 agent-council 妥协实现中抽象出来，成为独立主 Skill。

当前已经可以验收：

- 主仓结构成立。
- 主协议边界成立。
- Codex 主平台 foundation 和日常使用路径成立。
- Claude Code 入口、preflight、skill discovery、source-acquisition 只读片段和 pre-invocation dispatch decision 均已验证到当前阶段应有边界。
- 外部 skills 没有被复制进 core，而是通过 capabilities 和平台 adapter 管理。

## 验收命令

```bash
python3 checks/protocol_check.py
python3 checks/codex_validation_check.py
python3 checks/claude_code_validation_check.py
python3 checks/claude_dispatch_sample_check.py
python3 checks/claude_runtime_sample_check.py

```

其中 `claude_runtime_sample_check.py` 同时检查：

- 成功路径能生成 `sources[]` 和 `evidence[]`。
- 失败路径不伪造 `sources[]` 或 `evidence[]`。
- 失败路径必须说明未执行 fallback。

## 当前建议

下一阶段不要继续扩张协议层。

优先级应变为：

1. 实现 Claude Code 最小 adapter runtime。
2. 验证 result recovery 是否能自动生成 `sources[]` 和 `evidence[]`。
3. 再逐个接入 optional capability。
